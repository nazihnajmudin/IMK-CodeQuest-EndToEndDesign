# CodeQuest - IF3151 Human-Computer Interaction

## Project Overview

**CodeQuest** is a gamified mobile application designed for self-paced programming education, contributing to SDG 4 (Quality Education). The platform employs a Duolingo-like approach to coding education, transforming complex programming logic into bite-sized, interactive challenges that foster consistent learning habits.

### Key Features
- **Personalized Learning Paths** - Adaptive course recommendations based on user goals
- **Gamified Progression** - Daily quests, streaks, experience points, and achievement badges
- **Interactive Practice** - Hands-on coding exercises with contextual error guidance
- **Sandbox Environment** - Safe space for code experimentation and logic visualization
- **Community Forum** - Peer-to-peer discussion and knowledge sharing
- **Certificate System** - Verifiable credentials for professional portfolio

## Project Documentation

### Milestone 1: User Gathering

**Objective:** Conduct user research to understand target audience behaviors, motivations, and pain points in programming education.

**Activities:**
- Distributed online questionnaires (Google Form) to 32 respondents
- Analyzed learning habits, motivation types, and common obstacles
- Developed 6 distinct user personas representing target segments
- Identified 6 user goals and 17 functional tasks
- Established design challenges and usability/UX goals

**Key Findings:**
- 75% of users learn in the evening/night (night owls)
- 96.9% primarily use laptops, with significant smartphone adoption
- 84.4% motivated by formal recognition (certificates, level progression)
- Major obstacles: distraction, error frustration, and lack of guidance

**Deliverables:**
- User Persona profiles
- User Goals & Tasks matrix
- Design Challenges documentation
- Usability & UX Goals framework

### Milestone 3: Low-Fidelity Evaluation

**Objective:** Evaluate the low-fidelity prototype through user testing and questionnaire methods.

**Activities:**
- Designed 6 testing scenarios based on user goals
- Segmented 11 respondents into Experienced (54.5%) and Novice (45.5%) groups
- Conducted questionnaire-based evaluation (Likert scale 1-5)
- Analyzed feedback across three parameters: Information, Layout, Navigation

**Results Summary:**

| Parameter | Average Score |
|-----------|---------------|
| Information Quality | 4.54 |
| Layout Design | 4.60 |
| Navigation | 4.72 |
| **Overall Average** | **4.62 / 5.00** |

**Key Feedback & Improvements:**
- Added progress indicators for better orientation
- Enhanced daily streak visibility
- Restructured hint system (separated subtle hints from show-answer)
- Added responsive design considerations
- Improved terminology consistency (Sandbox → Experiment)

---

### Milestone 4: High-Fidelity Prototype

**Objective:** Transform wireframes into visually polished, interactive high-fidelity prototype.

**Activities:**
- Applied Material Design Guidelines
- Developed color system (primary: purple palette)
- Designed custom mascot and branding elements
- Implemented responsive layouts for mobile screens
- Created interactive Figma prototype with 28 screens
- Enhanced visual hierarchy and typography

**Key Design Elements:**

| Component | Design Approach |
|-----------|-----------------|
| Color System | Monochromatic purple palette with accent colors |
| Typography | Clear hierarchy with consistent sizing |
| Components | Rounded corners, subtle shadows, Material Design cards |
| Feedback | Visual indicators, progress bars, interactive states |
| Accessibility | High contrast, ergonomic touch targets |

**Major Improvements from Low-Fidelity:**
- Full branding implementation (logo, mascot, colors)
- Interactive code editor with syntax highlighting
- Rich text editor for community posts with toolbar
- Visual flowchart for sandbox execution
- Bottom sheet with contextual actions

---

### Milestone 5: High-Fidelity Evaluation

**Objective:** Conduct comprehensive usability testing of the high-fidelity prototype.

**Activities:**
- Evaluated 12 respondents (50% Experienced, 50% Novice)
- Applied Single Ease Question (SEQ) methodology
- Calculated System Usability Scale (SUS)
- Measured Net Promoter Score (NPS)
- Gathered qualitative feedback through 6 testing scenarios

**Results:**

| Metric | Score | Category |
|--------|-------|----------|
| SEQ Average | 74.33 / 100 | Good |
| SUS Score | 81.25 / 100 | Excellent |
| NPS Score | 41.66 | Good |

**Key Feedback & Planned Improvements:**

1. **User Onboarding:** Add interactive walkthrough for new users
2. **Terminology:** Simplify "Sandbox" → "Ruang Uji" / "Eksperimen"
3. **Daily Engagement:** Add daily pop-up questions/mini-challenges
4. **Community Access:** Consolidate to single navigation entry point
5. **Error Recovery:** Add bottom sheet for sandbox error analysis
6. **Certificate:** Maintain and enhance visual appeal
